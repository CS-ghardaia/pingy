## Brand & Style

The brand identity for Pingy revolves around **"Energy, Clarity, and Community."** It is tailored for young, tech-savvy Computer Science students who value modern design, speed, and collaborative learning. The visual direction merges **Bold Minimalism** with **Playful Confidence**, creating a digital environment that feels as dynamic as a student hackathon but as structured as a well-organized course syllabus.

The layout is **Arabic-first (RTL)**, ensuring that the natural flow of the script dictates the hierarchy and movement. The emotional response should be one of **"motivated focus"** — a clutter-free interface that reduces cognitive load while maintaining a youthful, high-energy SaaS aesthetic.

## Colors

The color strategy utilizes a **"Electric Yellow" (#F8C413)** as the primary anchor, symbolizing energy, creativity, and attention. This is balanced against a **deep Charcoal Black (#111111)** for text and structural elements to provide high contrast and legibility.

### Primary Colors
- **Primary Yellow (#F8C413):** Use for primary calls to action, active states, brand signatures, badges, and gamification highlights. This color demands attention without being aggressive.
- **Primary Black (#111111):** Use for primary text, headings, and dark mode backgrounds. Provides maximum contrast and authority.
- **Primary White (#FFFFFF):** Use for backgrounds, card surfaces, and text on dark backgrounds. Creates breathing room and cleanliness.

### Neutral Colors
- **Neutral 100 (#FEEFD0):** Warm cream — use for primary container backgrounds, hover states, and subtle highlights.
- **Neutral 200 (#D8D7D7):** Light gray — use for borders, dividers, and disabled states.
- **Neutral 300 (#6C6C6B):** Medium gray — use for secondary text, captions, and inactive icons.
- **Neutral 400 (#444443):** Dark gray — use for body text, labels, and secondary headings.

### Semantic Colors
- **Success (Emerald):** Use for positive feedback, approved content, completion states, and reputation gains.
- **Error (Crimson):** Use for validation errors, rejected content, deletion confirmations, and critical alerts.
- **Warning (Amber):** Use for pending states, moderation queues, and cautionary messages.
- **Info (Blue):** Use for informational banners, tips, and help text.

### Usage Rules
- Never use pure black (#000000) — always use #111111 for dark elements.
- Never use pure yellow (#FFFF00) — always use #F8C413 for brand consistency.
- Maintain a minimum contrast ratio of 4.5:1 for all text against backgrounds.
- Use the warm cream (#FEEFD0) sparingly — it should feel like a highlight, not a base.

## Typography

This design system is **Arabic-first and RTL-native**, designed primarily for Arabic academic workflows. However, given the Computer Science context, English code snippets and technical terms will be common.

### Font System

- **Manrope** → Used for:
  - All headings (H1, H2, H3, H4, H5)
  - Buttons and CTAs
  - Labels and UI emphasis elements
  - Body text (all weights: Regular, Bold, ExtraBold)

- **JetBrains Mono** (or Fira Code) → Used for:
  - Code snippets and blocks
  - Exercise problem statements with code
  - File names and technical identifiers
  - Console/output displays

### Design Philosophy

The system uses a single primary typeface (Manrope) for all UI text, creating a cohesive, modern feel. The geometric, friendly nature of Manrope aligns with the youthful energy of the student audience. For code-heavy content, a monospace font provides technical clarity.

### RTL Optimization

- Default alignment: **right**
- Line height:
  - Titles: 1.2 – 1.35
  - Body: 1.6 – 1.7
- Avoid tight letter spacing
- Maintain 8px spacing system
- Code blocks should be **LTR** with proper text direction handling

### Rules

- Never use Manrope ExtraBold for body text — reserve for headings and CTAs only.
- Never use Manrope Regular for buttons — always use Bold or ExtraBold.
- Code blocks should use JetBrains Mono at 14px with 1.6 line height.
- Do not mix fonts inside the same sentence — code terms in body text should use backticks, not font changes.

### Guidelines

- **Hierarchy:** Use font weight (800, 700, 400) rather than dramatic size jumps to distinguish between primary and secondary information.
- **Alignment:** Right-align all UI text by default; left-align code blocks and English technical terms within mixed-content areas.
- **Uppercase Labels:** Use uppercase with letter-spacing for small labels (tags, badges, navigation items) to create visual distinction.

## Layout & Spacing

The layout follows a **Fluid-Responsive** model. The platform utilizes a 12-column grid with 24px gutters to maintain a sense of openness and readability.

- **Margins:** Desktop views should maintain 48px external margins; mobile uses 16px.
- **Rhythm:** All vertical spacing must be a multiple of 8px.
- **RTL Flow:** The primary navigation resides on the right, with the content canvas expanding toward the left.
- **Content Density:** Keep information density moderate — students need focus, not overwhelm. Use generous whitespace between content cards.
- **Container Max Width:** 1280px to prevent excessive line lengths on large screens.

## Elevation & Depth

Depth is created through **subtle shadows** and **layered surfaces** rather than heavy borders.

- **Surfaces:** Use three tiers:
  1. Base (White #FFFFFF)
  2. Card (White with 8px soft shadow, 0 2px 8px rgba(17,17,17,0.06))
  3. Elevated (White with 16px soft shadow, 0 8px 24px rgba(17,17,17,0.08))
- **Shadows:** Shadows should be soft and diffused, using the Primary Black (#111111) tinted at 6-8% opacity for a clean, modern feel.
- **Overlays:** Modals and dropdowns must use a backdrop with 50% opacity black (#111111 at 50%) to maintain context while focusing the user.
- **Borders:** Use 1px borders with Neutral 200 (#D8D7D7) for card outlines; use 2px Primary Yellow (#F8C413) borders for focused or selected states.

## Shapes

The shape language is modern and approachable, reflecting the youthful energy of the brand.

- **Primary Containers:** Cards, modals, and panels use a consistent 16px radius (`rounded-lg`).
- **Small Elements:** Buttons and input fields use an 8px radius (`rounded-DEFAULT`) to maintain a tighter, more functional look.
- **Large Elements:** Hero sections and feature banners may go up to 24px (`rounded-xl`) for a softer, modern aesthetic.
- **Pills & Badges:** Use full radius (`rounded-full`) for tags, status badges, and category labels.
- **Iconography:** Use "Linear" icons with a 1.5pt stroke weight and rounded terminals to match the font geometry. Prefer outline icons over filled for a lighter feel.

## Animation & Motion

### Principles
- **Purposeful:** Every animation should guide the user or provide feedback
- **Fast:** Most transitions complete within 200-300ms
- **Smooth:** Use ease-out for entrances, ease-in-out for continuous motion

### Standard Transitions
- **Hover states:** 150ms ease-out
- **Focus states:** 200ms ease-out
- **Modal entrance:** 300ms ease-out, scale from 0.95 to 1, opacity 0 to 1
- **Modal exit:** 200ms ease-in, reverse of entrance
- **Page transitions:** 300ms ease-out, slide + fade
- **Toast notifications:** 400ms ease-out slide from top/bottom
- **Skeleton shimmer:** 1.5s linear infinite

### Micro-interactions
- Button press: Scale to 0.97 over 100ms
- Card hover: Lift (translateY -4px) + shadow increase over 200ms
- Badge unlock: Scale pulse (1 → 1.2 → 1) + glow effect
- New notification: Bell icon shake animation + yellow dot badge
- Content approval: Green checkmark draw animation + confetti burst (subtle)

## Accessibility

- All interactive elements must have minimum 44x44px touch targets
- Focus indicators: 2px solid #F8C413 outline, offset 2px
- Color is never the sole indicator of state — always pair with icons or text
- Respect `prefers-reduced-motion` — disable animations for users who request it
- All images must have descriptive alt text
- Form labels must be programmatically associated with inputs
- Error messages must be announced to screen readers
- Modal dialogs must trap focus and be dismissible with Escape key
